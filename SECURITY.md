# Security Policy

## Supported versions

Security fixes are applied to the latest commit on the default branch (`main`) unless a repository README states an explicit support matrix.

## Reporting a vulnerability

**Do not** open a public GitHub issue for exploitable security vulnerabilities.

Report privately to:

**Email:** dipak-kumar.singh@dsatechzone.com

Include:

- Description of the vulnerability and potential impact
- Steps to reproduce (proof of concept if available)
- Affected repository and component
- Your contact information for follow-up

## Response timeline

| Stage | Target |
|-------|--------|
| Acknowledgment | Within 48 hours |
| Triage and initial assessment | Within 5 business days |
| Remediation plan | Communicated after triage |

## Safe harbor

We support good-faith security research. Report issues responsibly and avoid accessing data beyond what is necessary to demonstrate the vulnerability.

A PGP public key will be published here when available. Until then, use the email address above.
