# Support

## How to get help

| Need | Channel |
|------|---------|
| Bug report | [Open a bug report issue](https://github.com/dsatechzone-solutions-hub/.github/issues/new?template=01_bug_report.yml) in the affected repository |
| Feature request | [Open a feature request](https://github.com/dsatechzone-solutions-hub/.github/issues/new?template=02_feature_request.yml) |
| Documentation gap | [Open a documentation issue](https://github.com/dsatechzone-solutions-hub/.github/issues/new?template=05_documentation.yml) |
| Design discussion (large change) | [Open an RFC](https://github.com/dsatechzone-solutions-hub/.github/issues/new?template=06_rfc.yml) |
| Security vulnerability | See [SECURITY.md](SECURITY.md) — **email only**, do not use public issues |
| General question | Open an issue in the relevant repo with label `type: task` or enable GitHub Discussions if available |

## Before opening an issue

1. Search existing issues in the target repository.
2. Read the repository README and [`CONTRIBUTING.md`](CONTRIBUTING.md).
3. Include reproduction steps, environment, and expected vs actual behavior for bugs.

## Engineering standards

All contributions must follow [`CONTRIBUTING.md`](CONTRIBUTING.md) and the pull request template.
