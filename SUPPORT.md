## How to get help

| Need | Channel |
|------|---------|
| Bug report | [![Bug report](https://img.shields.io/badge/template-bug%20report-d73a4a?style=flat-square)](https://github.com/dsatechzone-solutions-hub/.github/issues/new?template=01_bug_report.yml) |
| Feature request | [![Feature request](https://img.shields.io/badge/template-feature-a2eeef?style=flat-square)](https://github.com/dsatechzone-solutions-hub/.github/issues/new?template=02_feature_request.yml) |
| Documentation gap | [![Documentation](https://img.shields.io/badge/template-documentation-0075ca?style=flat-square)](https://github.com/dsatechzone-solutions-hub/.github/issues/new?template=05_documentation.yml) |
| Design discussion | [![RFC](https://img.shields.io/badge/template-RFC-d4c5f9?style=flat-square)](https://github.com/dsatechzone-solutions-hub/.github/issues/new?template=06_rfc.yml) |
| Security vulnerability | [![Email only](https://img.shields.io/badge/report-email%20only-critical?style=flat-square&logo=shield)](SECURITY.md) |
| General question | Issue with label `type: task` or GitHub Discussions (if enabled) |

Open issues in the **affected repository**, not only in this standards repo, unless the question is about org-wide templates.

## Before opening an issue

1. Search existing issues in the target repository.
2. Read the repository README and [CONTRIBUTING.md](CONTRIBUTING.md).
3. Include reproduction steps, environment, and expected vs actual behavior for bugs.

## Engineering standards

All contributions must follow [CONTRIBUTING.md](CONTRIBUTING.md) and the pull request template.
